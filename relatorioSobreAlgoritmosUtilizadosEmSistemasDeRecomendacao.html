Introdução¶

Este relatrio tem o objetivo de comparar algoritmos para a criao de sistemas de recomendao, para isto foi feito um sistema de recomendao para cada algoritmo analisado usando para isto uma base de dados de filmes, a base de dados utilizada foi a The Movies Dataset disponvel em: https://www.kaggle.com/rounakbanik/the-movies-dataset

A base de dados utilizada bastante rica na quantidade informao relacionada com os filmes presentes nela, possuindo muitos atributos, desta maneira foi realizada uma anlise exploratria da base de dados. O atributo da base de dados escolhido para ser utilizado pelos sistemas de recomendao desenvolvidos foi as notas que as pessoas do para os filmes.

Os sistemas de recomendao desenvolvidos, nos cdigos de programao presentes neste relatrios, so baseados nos algoritmos de RBM e correlao.

O RBM um algoritmo que usa redes neurais para fazer recomendao de filmes, o pacote RBM que foi utilizado para criao do sistema pode ser obtido em: https://github.com/meownoid/tensorfow-rbm

A correlao um algoritmo que diz o quanto duas variveis esto correlacionadas, foi usada a correlao de Person para criar o sistema de recomendao de filmes baseado em correlao.

Os arquivos utilizados para a elaborao deste relatrio podem ser obtidos em: http://rebrand.ly/2019d

Carregamento da base de dados, filtragem e transformação dos dados¶

A base de dados esta divida entre vrios arquivos CSV. Apenas dois arquivos CSVs da base foram utilizados neste relatrio: ratings_small.csv e movies_metadata.csv.

O arquivo movies_metadata.csv possui informaes diversas sobre os filmes, entre as informaes esto: oramento, data de lanamento, a mdia sobre notas que o filme recebeu e o nmero de pessoas que deram notas.

O arquivo ratings_small.csv possui a informao sobre alguns usurios que deram notas para alguns filmes, este arquivo pode ser considerado uma verso menor do arquivo ratings.csv. O uso do ratings_small.csv ao invs do ratings.csv porque o ratings.csv possui muitos dados, isto torna o uso do ratings.csv computacionalmente mais custoso para as tarefas descritas neste relatrio.

O dados existentes no arquivo movies_metadata.csv foram utilizados na anlise exploratria e os dados existentes em ratings_small.csv foram utilizados tanto para analise exploratria quanto para criao dos sistemas recomendao. Um detalhe importante que as notas que usurios do para os filmes em ratings_small.csv esta em uma escala de 0 a 5 enquanto que a nota mdia que filmes possuem em movies_metadata.csv esta em uma escala de 0 a 10, para evitar confuses a nota mdia que os usurios do para os filmes foi escalonada para 0 a 5 nos grficos e tabelas presentes neste relatrio que fazem uso deste dado.

Criação dos sistemas de recomendação¶

Os sistemas vo ser criados a partir do RBM e da correlao, alm disto vai ser criado um sistema de recomendao mais simples que recomenda filmes aleatoriamente. Os dados contidos no arquivo ratings_small.csv foram utilizado para criar os sistemas, estes dados so linhas de uma matriz e cada linha contem o id (nmero de identificao) do filme, o id da pessoa e a nota que a pessoa deu para o filme. A seguir uma amostra dos dados contidos no arquivo ratings_small.csv:

De todas linhas em ratings_small.csv 70% delas foram escolhidas para serem parte da base de treinamento dos sistemas e 30% foram escolhidas para serem parte da base de testes dos sistemas, a escolha de que linhas foram usadas para treinamento ou para teste foi feito de maneira aleatria.

As notas contidas em ratings_small.csv variam de 0 a 5, contudo o sistema criado a partir do RBM s pode aceitar em seu treinamento notas que tem o valor 0 (no viu ou no gostou do filme) ou o valor 1 (gostou do filme). Por isto foi considerado que pessoas que deram uma nota maior que 4 gostaram do filme (valor 1) e pessoas que deram uma nota menor no gostaram (valor 0).

Para diminuir custo computacional de treinamento dos sistemas apenas uma parte dos filmes foram considerados no treinamento e teste dos sistemas, a escolha dos filmes que fazem parte deste conjunto baseado no quo bem avaliado pelas pessoas um filme considerando apenas os dados da base de treinamento. Se um filme que receber uma nota maior que 4 de uma pessoa significa que a pessoa deu uma boa avaliao para o filme, ou seja a pessoa deu um like para o filme. Os 1024 filmes com maior nmero de likes foram selecionados para fazerem parte dos filmes que so utilizados para treinar os sistemas. importante observar que um sistema s pode recomendar filmes com quais tenha sido treinado.

Teste dos sistemas de recomendação¶

Os sistemas criados a partir do RBM e da correlao foram comparados com um sistema simples que recomenda aleatoriamente 14 filmes entre os 20 filmes com mais likes para uma pessoa assistir.

A entrada para todos os sistema de recomendao um vetor binrio com 1024 elementos onde cada elemento no vetor corresponde a um filme, este vetor representa os filmes que uma dada pessoa assistiu e gostou. Se o elemento do vetor tiver valor 1 significa que a pessoa gostou daquele filme se tiver valor 0 significa que a pessoa no viu ou no gostou do filme.

A sada que os sitemas de recomendao devolvem um vetor 1024, assim como a entrada cada elemento deste vetor corresponde a um filme e um elemento do vetor com 1 significa que pessoa vai gostar do filme, um elemento com 0 significa que a pessoa no vai gostar do filme. Uma coisa a ser observar que todos os sistemas criados podem recomendar filmes que as pessoas j assistiram e gostaram.

Análise sobre os sistemas criados¶

Observando os dados pode-se notar que o melhor sitemas foi aquele criado a partir do RBM, entretanto a acurcia do RBM se aproxima muito do sistema aleatrio de recomendao e este sitema apresenta uma acurcia superior ao sistema criado por correlao. Uma explicao para o sistema aleatrio de recomendao ter tido uma acurcia comparvel ao RBM e superior ao de correlao a pequena quantidade de dados com quais os sistemas foram treinados, talvez utilizando os dados existentes em ratings.csv, ao invs dos dados de ratings_small.csv, sistemas de recomendao com maior acurcia fossem criados. O motivo de no utilizar os dados existentes em ratings.csv para treinar os sistemas que isto demandaria muito tempo de treino pois o arquivo ratings.csv tem 260 vezes mais dados que o ratings_small.csv.

Conclusão¶

Este relatrio apresentou uma anlise exploratria e uma comparao entre sistemas de recomendao para a base de dados The Movies Dataset.

A anlise exploratria mostrou que filmes tendem a serem lanados com mais frequncia na sexta-feira e que no sculo XX filmes eram lanados com mais frequncia em Julho algo que mudou no seculo XXI onde setembro o ms com mais lanamentos de filmes.

A comparao entre sistemas de recomendao comparou o acurcia de trs sistemas, um criado a partir do algoritmo RBM, outro criado por correlao e um ltimo mais simples que recomenda aleatoriamente 14 filmes entre os 20 mais populares. O RBM foi o sistema com maior acurcia, entretanto sua acurcia ficou muito prxima da acurcia obtida pelo sistema que faz recomendaes aleatrias de filmes, o sistema com pior acurcia foi o sistema criado com base em correlao. Uma das possveis causas para o sistema que faz recomendaes aleatrias ter uma acurcia comparvel ao RBM e superior ao sistema por correlao pode ser a pouca quantidade de dados com quais o RBM e o sistema por correlao foram treinados.
