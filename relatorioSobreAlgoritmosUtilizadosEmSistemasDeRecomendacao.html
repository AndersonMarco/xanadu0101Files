Introdução¶

Este relatório tem o objetivo de comparar algoritmos para a criação de sistemas de recomendação, para isto foi feito um sistema de recomendação para cada algoritmo analisado usando para isto uma base de dados de filmes, a base de dados utilizada foi a The Movies Dataset disponível em: https://www.kaggle.com/rounakbanik/the-movies-dataset

A base de dados utilizada é bastante rica na quantidade informação relacionada com os filmes presentes nela, possuindo muitos atributos, desta maneira foi realizada uma análise exploratória da base de dados. O atributo da base de dados escolhido para ser utilizado pelos sistemas de recomendação desenvolvidos foi as notas que as pessoas dão para os filmes.

Os sistemas de recomendação desenvolvidos, nos códigos de programação presentes neste relatórios, são baseados nos algoritmos de RBM e correlação.

O RBM é um algoritmo que usa redes neurais para fazer recomendação de filmes, o pacote RBM que foi utilizado para criação do sistema pode ser obtido em: https://github.com/meownoid/tensorfow-rbm

A correlação é um algoritmo que diz o quanto duas variáveis estão correlacionadas, foi usada a correlação de Person para criar o sistema de recomendação de filmes baseado em correlação.

Os arquivos utilizados para a elaboração deste relatório podem ser obtidos em: http://rebrand.ly/2019d

Carregamento da base de dados, filtragem e transformação dos dados¶

A base de dados esta divida entre vários arquivos CSV. Apenas dois arquivos CSVs da base foram utilizados neste relatório: ratings_small.csv e movies_metadata.csv.

O arquivo movies_metadata.csv possui informações diversas sobre os filmes, entre as informações estão: orçamento, data de lançamento, a média sobre notas que o filme recebeu e o número de pessoas que deram notas.

O arquivo ratings_small.csv possui a informação sobre alguns usuários que deram notas para alguns filmes, este arquivo pode ser considerado uma versão menor do arquivo ratings.csv. O uso do ratings_small.csv ao invés do ratings.csv é porque o ratings.csv possui muitos dados, isto torna o uso do ratings.csv computacionalmente mais custoso para as tarefas descritas neste relatório.

O dados existentes no arquivo movies_metadata.csv foram utilizados na análise exploratória e os dados existentes em ratings_small.csv foram utilizados tanto para analise exploratória quanto para criação dos sistemas recomendação. Um detalhe importante é que as notas que usuários dão para os filmes em ratings_small.csv esta em uma escala de 0 a 5 enquanto que a nota média que filmes possuem em movies_metadata.csv esta em uma escala de 0 a 10, para evitar confusões a nota média que os usuários dão para os filmes foi escalonada para 0 a 5 nos gráficos e tabelas presentes neste relatório que fazem uso deste dado.

Sistemas de recomendação¶

Criação dos sistemas de recomendação¶

Os sistemas vão ser criados a partir do RBM e da correlação, além disto vai ser criado um sistema de recomendação mais simples que recomenda filmes aleatoriamente. Os dados contidos no arquivo ratings_small.csv foram utilizado para criar os sistemas, estes dados são linhas de uma matriz e cada linha contem o id (número de identificação) do filme, o id da pessoa e a nota que a pessoa deu para o filme. A seguir uma amostra dos dados contidos no arquivo ratings_small.csv:

De todas linhas em ratings_small.csv 70% delas foram escolhidas para serem parte da base de treinamento dos sistemas e 30% foram escolhidas para serem parte da base de testes dos sistemas, a escolha de que linhas foram usadas para treinamento ou para teste foi feito de maneira aleatória.

As notas contidas em ratings_small.csv variam de 0 a 5, contudo o sistema criado a partir do RBM só pode aceitar em seu treinamento notas que tem o valor 0 (não viu ou não gostou do filme) ou o valor 1 (gostou do filme). Por isto foi considerado que pessoas que deram uma nota maior que 4 gostaram do filme (valor 1) e pessoas que deram uma nota menor não gostaram (valor 0).

Para diminuir custo computacional de treinamento dos sistemas apenas uma parte dos filmes foram considerados no treinamento e teste dos sistemas, a escolha dos filmes que fazem parte deste conjunto é baseado no quão bem avaliado pelas pessoas um filme é considerando apenas os dados da base de treinamento. Se um filme que receber uma nota maior que 4 de uma pessoa significa que a pessoa deu uma boa avaliação para o filme, ou seja a pessoa deu um like para o filme. Os 1024 filmes com maior número de likes foram selecionados para fazerem parte dos filmes que são utilizados para treinar os sistemas. É importante observar que um sistema só pode recomendar filmes com quais tenha sido treinado.

Teste dos sistemas de recomendação¶

Os sistemas criados a partir do RBM e da correlação foram comparados com um sistema simples que recomenda aleatoriamente 14 filmes entre os 20 filmes com mais likes para uma pessoa assistir.

A entrada para todos os sistema de recomendação é um vetor binário com 1024 elementos onde cada elemento no vetor corresponde a um filme, este vetor representa os filmes que uma dada pessoa assistiu e gostou. Se o elemento do vetor tiver valor 1 significa que a pessoa gostou daquele filme se tiver valor 0 significa que a pessoa não viu ou não gostou do filme.

A saída que os sitemas de recomendação devolvem é um vetor 1024, assim como a entrada cada elemento deste vetor corresponde a um filme e um elemento do vetor com 1 significa que pessoa vai gostar do filme, um elemento com 0 significa que a pessoa não vai gostar do filme. Uma coisa a ser observar é que todos os sistemas criados podem recomendar filmes que as pessoas já assistiram e gostaram.

def countNumberOfElementsEqualsForTwoVectorsThatAreEqualToTrue(v1,v2):
    count=0
    for i in range(len(v1)):

def testModel (users,model):
    informationReturned={}
    numberOfElementsRecommendedThatTheUserWillViewList=[]
    numberOfElementsThatTheUserWillViewList=[]
    percentageForHitRecommendationsList=[]
    percentageForWrongRecommendationsList=[]
    iteration=0    
    nUsers=len(users)
    numberOfElementsRecommendedList=[]
    for k in users:
        user=users[k]


        #Each position of the vector "elementsRecomended" represent an movieId
        # the dics convertermovieIdToVectorsPositionOfModels and 
        #converterVectorsPositionOfModelsTomovieId has the information about
        #the movieid and the position inside vector that represents the movieid.
        #If the model predicts that the user wiil view and like of the movie the 
        #vector element that is pointed by movieid of the movie is equal to 1,
        #if predicts that the user will not view or will not like of the movie the 
        #vector element that is pointed by movieid of the movie is equal to 0.
        elementsRecomended=model(user["movieListToPassAsInputForTheModel"])
        elementsRecomendedCopy=np.copy(elementsRecomended)
        elementsRecomended=(elementsRecomended^user["movieListToPassAsInputForTheModel"])&elementsRecomendedCopy
        

        numberOfElementsThatTheUserWillView=sum(user["movieListToTestModel"]>0.5)
        numberOfElementsRecommendedThatTheUserWillView=countNumberOfElementsEqualsForTwoVectorsThatAreEqualToTrue(user["movieListToTestModel"]>0.5,elementsRecomended)
        numberOfElementsRecommended=sum(elementsRecomended)
        numberOfElementsRecommendedList.append(numberOfElementsRecommended)
        numberOfElementsRecommendedThatTheUserWillViewList.append(numberOfElementsRecommendedThatTheUserWillView)
        numberOfElementsThatTheUserWillViewList.append(numberOfElementsThatTheUserWillView)
        if(numberOfElementsRecommended>0.0):
            percentageForHitRecommendationsList.append(numberOfElementsRecommendedThatTheUserWillView/numberOfElementsRecommended)
        if(numberOfElementsRecommended>0.0):
            percentageForWrongRecommendationsList.append((numberOfElementsRecommended-numberOfElementsRecommendedThatTheUserWillView)/numberOfElementsRecommended)

        #if((iteration%1000)==0):
        #    print(str(iteration)+"|"+str(nUsers))
        iteration=iteration+1
    informationReturned['numberOfElementsRecommendedThatTheUserWillView']={'avg':np.mean(numberOfElementsRecommendedThatTheUserWillViewList),'std':np.std(numberOfElementsRecommendedThatTheUserWillViewList)}
    informationReturned['numberOfElementsThatTheUserWillViewList']={'avg':np.mean(numberOfElementsThatTheUserWillViewList),'std':np.std(numberOfElementsThatTheUserWillViewList)}
    informationReturned['percentageForHitRecommendationsList']={'avg':np.mean(percentageForHitRecommendationsList),'std':np.std(percentageForHitRecommendationsList)}
    informationReturned['percentageForWrongRecommendationsList']={'avg':np.mean(percentageForWrongRecommendationsList),'std':np.std(percentageForWrongRecommendationsList)}
    informationReturned['numberOfElementsRecommended']={'avg':np.mean(numberOfElementsRecommendedList),'std':np.std(numberOfElementsRecommendedList)}
    return informationReturned

Análise sobre os sistemas criados¶

Observando os dados pode-se notar que o melhor sitemas foi aquele criado a partir do RBM, entretanto a acurácia do RBM se aproxima muito do sistema aleatório de recomendação e este sitema apresenta uma acurácia superior ao sistema criado por correlação. Uma explicação para o sistema aleatório de recomendação ter tido uma acurácia comparável ao RBM e superior ao de correlação é a pequena quantidade de dados com quais os sistemas foram treinados, talvez utilizando os dados existentes em ratings.csv, ao invés dos dados de ratings_small.csv, sistemas de recomendação com maior acurácia fossem criados. O motivo de não utilizar os dados existentes em ratings.csv para treinar os sistemas é que isto demandaria muito tempo de treino pois o arquivo ratings.csv tem 260 vezes mais dados que o ratings_small.csv.

Conclusão¶

Este relatório apresentou uma análise exploratória e uma comparação entre sistemas de recomendação para a base de dados The Movies Dataset.

A análise exploratória mostrou que filmes tendem a serem lançados com mais frequência na sexta-feira e que no século XX filmes eram lançados com mais frequência em Julho algo que mudou no seculo XXI onde setembro é o mês com mais lançamentos de filmes.

A comparação entre sistemas de recomendação comparou o acurácia de três sistemas, um criado a partir do algoritmo RBM, outro criado por correlação e um último mais simples que recomenda aleatoriamente 14 filmes entre os 20 mais populares. O RBM foi o sistema com maior acurácia, entretanto sua acurácia ficou muito próxima da acurácia obtida pelo sistema que faz recomendações aleatórias de filmes, o sistema com pior acurácia foi o sistema criado com base em correlação. Uma das possíveis causas para o sistema que faz recomendações aleatórias ter uma acurácia comparável ao RBM e superior ao sistema por correlação pode ser a pouca quantidade de dados com quais o RBM e o sistema por correlação foram treinados.
