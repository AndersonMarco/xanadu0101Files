Introdução¶

Este relatrio tem objetivo de analisar qual modelo melhor para a predio de fraudes em uma base de transaes bancarias. Os modelos analisados foram gerados pelos cdigos presentes neste relatrio a partir dos algoritmos de aprendizado de mquina:

  • rvore de Deciso
  • Rede Neural
  • Regresso Logstica

Devido ao tempo necessrio para criar os modelos eles foram guardados em uma base de dados SQLite, esta base se encontra no arquivo databaseFormodelsCreated.sqlite3 que esta dentro do arquivo compactado que possui o cdigo fonte deste relatrio. O arquivo databaseFormodelsCreated.sqlite3 pode ser aberto e manipulados por programas especficos, como por exemplo o SQL Lite Studio que pode ser obtido gratuitamente em https://sqlitestudio.pl/index.rvt.

Uma caracterstica importante sobre a base de dados analisada que as classes dela esto muito desbalanceadas, mais de 95% das transaes da base so da classe legtima, deste modo analisar os modelos com base na acurcia simples no uma coisa que pode ser feita.

A base de transaes bancarias utilizada pode ser encontrada no site: https://www.kaggle.com/mlg-ulb/creditcardfraud/home

O cdigo fonte para este relatrio pode ser baixado em: http://bit.ly/2DgBzYC

Geração dos modelos¶

Os modelos criados esto divididos em 4 tipos diferentes de algoritmos de aprendizado de mquina. Dezesseis modelos foram criados utilizando a tcnica de validao cruzada para os algoritmos, os modelos foram salvos no arquivo databaseFormodelsCreated.sqlite3. A tcnica de validao cruzada consiste em dividir a base de dados em K partes (neste caso K=4) e criar K modelos para cada algoritmo, em cada algoritmo utilizada, em alternncia, K-1 partes para treinamento e uma parte para teste.

def trainModelsOfTypeKeras(clfFactory,X,Y):
    #clfFactory             => Function to create classifier object
    #X                      => Elements attributes  in dataset
    #Y                      => Elements classes in dataset
    kf = KFold(n_splits=4,shuffle=True,random_state=3223)    
    foldsProcessed=0    
    vectWithTheTestIndexesForEachFold=[]
    modelsForEachFold=[]
    modelsType=[]
    #parametersToCompileAndFitTheModel===========================================
    parametersForTheModel={
                            "parametersToFit":{"batch_size":100,"class_weight":{0: 1.,1: 50.,},"epochs":1},
                            "parametersToCompile":{ "optimizer":'adam',"loss":'sparse_categorical_crossentropy','metrics':['accuracy']}
    }
    #end=========================================================================

    for train_index, test_index in kf.split(X):
        print("folds processed:"+str(foldsProcessed))
        foldsProcessed=foldsProcessed+1
        X_train  = X.loc[train_index]
        Y_train  = Y[train_index]
        vectWithTheTestIndexesForEachFold.append(test_index)        
        clf=clfFactory()        
        
        clf.compile(optimizer=parametersForTheModel["parametersToCompile"]["optimizer"],
                    loss=parametersForTheModel["parametersToCompile"]["loss"],
                    metrics=parametersForTheModel["parametersToCompile"]["metrics"])
        clf.fit(X_train.values,Y_train.values,
                batch_size=parametersForTheModel["parametersToFit"]["batch_size"],
                class_weight=parametersForTheModel["parametersToFit"]["class_weight"],
                epochs=parametersForTheModel["parametersToFit"]["epochs"])        
        modelsForEachFold.append(clf)
        modelsType.append(neuralNetworkFromKeras)         
    return {            
            "vectWithTheTestIndexesForEachFold":vectWithTheTestIndexesForEachFold,
            "modelsForEachFold":modelsForEachFold,
            "parametersForTheModel":parametersForTheModel,
            "modelsType": modelsType

#load models trained=========================================================
import sqlite3
import json
import base64
import pickle
from keras.models import model_from_json, load_model
from keras.initializers import glorot_uniform
from keras.utils import CustomObjectScope
def loadKerasModel(modelStoredAtBase64, parametersToCompileModel):
    #https://stackoverflow.com/questions/53183865/unknown-initializer-glorotuniform-when-loading-keras-model
    modelTrained=None
    with CustomObjectScope({'GlorotUniform': glorot_uniform()}):
        output_file = open('tempFileForModel.hdf5', 'wb')
        output_file.write(base64.b64decode(modelStoredAtBase64))
        output_file.close()
        modelTrained=load_model('tempFileForModel.hdf5')
        modelTrained.compile(optimizer=parametersToCompileModel["optimizer"],
                             loss=parametersToCompileModel["loss"],
                             metrics=parametersToCompileModel["metrics"])
    return modelTrained
    
def loadScikitModel(modelStoredAtBase64):
    fp = open('tempFileForModel.sav', 'wb')
    fp.write(base64.b64decode(modelStoredAtBase64))
    fp.close()
    fp=open('tempFileForModel.sav', 'rb')
    model=pickle.load(fp)
    fp.close()
    return model

conn = sqlite3.connect('databaseFormodelsCreated.sqlite3')
cur=conn.cursor()
trainingSessions=[]
idsForTrainingSessions=[]
descriptionsForTrainingSessions=[]
cur.execute('SELECT id,description FROM training_session  ORDER BY id ASC LIMIT 40')
rows=cur.fetchall()
for row in rows:
    idsForTrainingSessions.append(row[0])
    descriptionsForTrainingSessions.append(row[1])

for i in range(len(idsForTrainingSessions)):    
    cur.execute('SELECT mo.model_stored_at_base64, param.parameters, mo.vect_with_the_test_indexes_at_dataframe, mo.type, mo.id, ty.name FROM model as mo  inner join model_parameters as param  on mo.model_parameters =param.id  inner join type_of_model as ty on ty.id=mo.type  where mo.training_session={idForTrainingSession} ORDER BY mo.id  ASC LIMIT 40'.format(idForTrainingSession=idsForTrainingSessions[i]))
    rows = cur.fetchall()
    modelsTrainedLoadedFromDisk=[]
    for row in rows:    
        if(row[3]==neuralNetworkFromKeras):
            modelLoaded=loadKerasModel(row[0],json.loads(row[1])['parametersToCompile'])
        else:
            modelLoaded=loadScikitModel(row[0])

        modelsTrainedLoadedFromDisk.append({"modelTrained": modelLoaded, "vectWithTheTestIndexesAtDataFrame":json.loads(row[2]), "type":row[3],'idForModelInDatabase':row[4],'description':row[5]})
    
    trainingSessions.append({'modelsTrained':modelsTrainedLoadedFromDisk,'description':descriptionsForTrainingSessions[i],'id':idsForTrainingSessions[i]})

conn.close()
#end=========================================================================

Análise dos modelos¶

Para a anlise as transaes bancarias foram dvidas em 6 tipos:

  • Transaes fraudulentas.
  • Transaes legtimas.
  • Transaes que so da classe legtima e que o modelo previu como legtimas.
  • Transaes que so da classe legtima mas que o modelo previu como fraudulentas.
  • Transaes que so da classe fraudulenta mas que o modelo previu como legtimas.
  • Transaes que so da classe fraudulenta e que o modelo previu como fraudulentas.

Na linha de cada matriz mostrada a seguir esta um modelo e na coluna esta o tipo de transao. As matrizes mostradas a seguir foram feitas com base nos dados obtidos pelos elementos de testes dos modelos, estes elementos so obtidos da base de dados original (o base guardada no arquivo *.csv).

Para fim de validade um modelo no pode ser testado com um conjunto de dados que tenha elementos que foram usados para treinar o modelo, na validao cruzada (que foi a metodologia utilizada para gerar os modelos) os conjuntos de dados escolhidos para testar um modelo so usados para treinar outro modelo, portanto cada modelo possui conjuntos de testes diferentes. Deste modo a soma das linhas das matrizes a seguir nem sempre vai ser igual.

Análise dos dados obtidos para a escolha do modelo de previsão¶

Observando os dados levantados com relao os modelos de previso pode-se concluir que o melhor modelo de previso o com id igual a 718, os motivos para esta escolha so:

  • A quantidade de transaes fraudulentas no detectadas a mais baixa que existe. Este fato por ser observado na Subseo Anlise para quantidade de tipos de transaes

  • A quantidade de dinheiro a ser bloqueado pelo modelo por este achar de forma errada que a transao era fraudulenta quando na verdade era legtima tambm a mais baixa que existe. Este fato por ser observado na Subseo Matriz mostrando a quantidade de dinheiro movimentado pelos tipos de transaes.

  • A quantidade de dinheiro movimentado por transaes fraudulentas no detectadas pela modelo tambm a mais baixa que existe. Este fato por ser observado tambm na Subseo Matriz mostrando a quantidade de dinheiro movimentado pelos tipos de transaes.

  • As transaes que o modelo bloquearia erroneamente, por achar que transao era fraudulenta quando na verdade era legitima, so transaes de baixo valor. Por serem de baixo valor este bloqueio causaria poucos transtornos para os envolvidos nas transaes, os envolvidos poderiam realizar a transao por outra forma de pagamento (como por exemplo dinheiro em espcie). Este dado por ser observado na Subseo Histogramas mostrando a quantidade de dinheiro movimentado pelos tipos de transaes para cada modelo.

Um detalhe importante foi que ao analisar os modelos gerados, utilizando como critrios de anlise os motivos apresentados antes, os modelos de predio baseados em rede neural so os que apresentam os melhores resultados.
